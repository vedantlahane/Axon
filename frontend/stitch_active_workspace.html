To optimize this, we should avoid multiple joins on large datasets and instead utilize a Window Function. This allows us to partition the data efficiently in a single scan of the orders table.

description optimized_query.sql 
WITH CustomerTotals AS (
    SELECT 
        customer_id,
        SUM(amount) OVER(PARTITION BY customer_id) as total_spent
    FROM orders
    WHERE order_date >= CURRENT_DATE - 30
),
RankedOrders AS (
    SELECT *,
        ROW_NUMBER() OVER(
            PARTITION BY customer_id 
            ORDER BY order_date DESC
        ) as rn
    FROM orders
    WHERE customer_id IN (
        SELECT customer_id FROM CustomerTotals WHERE total_spent > 500
    )
)
SELECT * FROM RankedOrders WHERE rn = 1;

By using ROW_NUMBER(), we reduce the complexity to O(n) for the filtering pass. With a proper index on (customer_id, order_date), this will execute in milliseconds.
