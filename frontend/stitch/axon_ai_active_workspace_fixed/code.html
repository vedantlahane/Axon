WITH UserSpending AS (
  SELECT customer_id
  FROM orders
  WHERE order_date >= CURRENT_DATE - INTERVAL '30 days'
  GROUP BY customer_id
  HAVING SUM(amount) > 500
),
LatestOrders AS (
  SELECT *,
    ROW_NUMBER() OVER (PARTITION BY customer_id ORDER BY order_date DESC) as rank
  FROM orders
  WHERE customer_id IN (SELECT customer_id FROM UserSpending)
)
SELECT * FROM LatestOrders WHERE rank = 1;

To optimize this, I implemented a CTE (Common Table Expression) to filter high-value customers first. This reduces the dataset before applying the expensive ROW_NUMBER() window function.

By narrowing down the customer_id space in the first pass, we leverage existing indexes on order_date and customer_id, which should reduce execution time from 2s to sub-200ms.
